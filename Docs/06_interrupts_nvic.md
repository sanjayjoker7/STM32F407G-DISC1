# STM32F407 Interrupt System & NVIC (Deep Dive)

## 1. Introduction

Interrupts allow the CPU to respond immediately to events without continuously polling.

In STM32F407 (Cortex-M4), interrupts are managed by:

NVIC – Nested Vector Interrupt Controller

The system is fully hardware-supported for fast context switching and deterministic real-time behavior.

---

## 2. Cortex-M4 Exception Model

Interrupts in Cortex-M4 are called "exceptions".

There are two main categories:

1. System Exceptions (core-level)
2. External Interrupts (peripheral-generated)

---

## 3. System Exceptions (Core Exceptions)

These are built into the Cortex-M4 core.

| Exception        | Description |
|------------------|-------------|
| Reset            | System startup |
| NMI              | Non-maskable interrupt |
| HardFault        | Critical fault |
| MemManage        | MPU fault |
| BusFault         | Bus access fault |
| UsageFault       | Illegal instruction fault |
| SVCall           | Supervisor call |
| PendSV           | Context switching (RTOS) |
| SysTick          | System timer |

HardFault is the most common debugging issue.

---

## 4. External Interrupts (Peripheral IRQs)

Generated by STM32 peripherals:

- EXTI (GPIO interrupts)
- TIM (Timers)
- USART
- SPI
- I2C
- ADC
- DMA
- USB
- CAN

Each peripheral has its own IRQ number.

Example:
EXTI0_IRQn → External interrupt line 0

---

## 5. Vector Table

Located at:

0x08000000 (Flash default)

Structure:

| Offset | Purpose |
|--------|----------|
| 0x00   | Initial Stack Pointer |
| 0x04   | Reset Handler |
| 0x08   | NMI Handler |
| 0x0C   | HardFault Handler |
| ...    | Other ISRs |

Each entry stores the address of the ISR.

When interrupt occurs:
CPU fetches ISR address from vector table.

---

## 6. Hardware Interrupt Flow

Step-by-step:

1. Event occurs
2. Interrupt flag set in peripheral
3. NVIC detects enabled interrupt
4. CPU:
   - Completes current instruction
   - Pushes registers onto stack
5. Jumps to ISR
6. ISR executes
7. Return from interrupt
8. Registers restored
9. Main program resumes

---

## 7. Automatic Context Saving

On interrupt entry, Cortex-M4 automatically pushes:

- R0
- R1
- R2
- R3
- R12
- LR
- PC
- xPSR

This makes interrupt handling fast and efficient.

---

## 8. NVIC Priority System

STM32F407 supports configurable interrupt priorities.

Lower priority number = Higher priority.

Example:

Priority 0 → Highest  
Priority 15 → Lowest  

Interrupt nesting allows:

High-priority interrupt to interrupt a lower-priority ISR.

---

## 9. Preemption vs Subpriority

Priority grouping allows:

- Preemption priority
- Subpriority

Preemption → Can interrupt another ISR  
Subpriority → Order if same preemption level

Configured using:

HAL_NVIC_SetPriority()

---

## 10. Enabling Interrupt (HAL Example)

Example: Enable EXTI0 interrupt

```c
HAL_NVIC_SetPriority(EXTI0_IRQn, 1, 0);
HAL_NVIC_EnableIRQ(EXTI0_IRQn);
